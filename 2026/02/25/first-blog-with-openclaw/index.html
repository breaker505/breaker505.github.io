把博客重新整理一遍，也把 OpenClaw 接进了我的写作流程

这篇文章算是我重新整理个人博客后的第一篇正式记录。

过去这个博客更像是一个“先搭起来再说”的站点，能打开、能发文章，但还谈不上真正属于我自己。主题是默认的，结构是松散的，分类和标签没有收拢，很多地方都停留在“能用”而不是“愿意长期维护”的状态。

所以这次我决定认真把它整理一遍。不是为了做一个多复杂的网站，而是想把这里慢慢变成一个真正长期使用的个人空间。更重要的是，我第一次把 OpenClaw 真正接进了这个过程里，让它不只是回答问题，而是开始参与我的博客写作和整理流程。

为什么重新整理博客

我一直觉得，博客不只是发文章的地方，它更像是一个人长期积累的外部记忆。

很多东西如果只停留在聊天、零散笔记或者某次一闪而过的想法里，过不了多久就会散掉。尤其是技术学习这件事，学过、做过、踩过坑，如果没有留下结构化记录，最后往往只剩一个模糊印象。

所以这次整理博客，本质上是在整理我过去留下来的内容，也是在给以后继续写作打地基。

我希望它至少能做到几件事：

  • 页面风格更统一，不再像一个默认模板站点
  • 分类和标签更克制，后面写文章不会越来越乱
  • 能同时容纳技术内容和个人思考
  • 以后写新文章时，流程足够顺，不需要每次都重新折腾环境

这次做了哪些调整

这次我主要做了几件事情。

第一，重新整理主题和页面风格

博客原来使用的是另一套主题，虽然能正常用，但整体感觉还不够像“我自己的地方”。后来我把主题切换到了更干净的 Fluid，也顺手把首页、关于页、导航栏、页脚这些位置都做了私人化调整。

比如：

  • 博客名称改成了“鱼跃厂长”
  • 首页文案重新写过
  • 换了新的背景图
  • 关于页改成了更像个人主页的风格

这些改动不大，但效果很直接。至少现在打开首页的时候，不再是“某个默认博客模板”，而更像一个真的会长期维护下去的个人站点。

第二，把博客写作链路重新跑通

这次还有一个对我来说挺特别的点，就是我没有完全手动折腾所有细节，而是尝试把 OpenClaw 真正接入到博客整理这件事里。

从一开始确认博客目录、识别 Hexo 结构、测试文章写入，到后面修依赖、修主题配置、切换主题、部署上线，整个过程都在逐步摸清楚：

OpenClaw 能不能真正参与我的个人工作流。

事实证明，这条路是能跑通的。

它不只是“帮我回答问题”，而是真的能参与一部分具体工作，比如：

  • 确认 Hexo 文章目录
  • 直接写 Markdown 文件
  • 修改主题配置
  • 协助调整页面文案
  • 生成并部署站点
  • 帮我一起整理分类和标签体系

这件事对我来说挺有意义，因为它意味着以后写博客这件事，可以不只是“我一个人从头折腾到尾”，而是能把一部分整理、润色、结构化的工作交给一个稳定的工具链来配合完成。

为什么我想把 OpenClaw 放进这个过程里

我最近也在学习 Agent 相关的东西，所以这次整理博客，对我来说其实不只是一次站点维护。

它更像是一个很小但很真实的实验：

如果一个 Agent 不只是聊天，而是真的接入我的个人工作流，它能不能在一些长期重复但又需要理解上下文的事情上发挥作用？

博客写作刚好是一个很合适的场景。

因为它同时具备几个特点：

  • 有固定目录结构
  • 有比较明确的输出格式
  • 需要理解上下文和写作意图
  • 又不只是机械替换文本

这比单纯问答更接近“协作”。

而这次从整理旧博客到调主题、修图片、发文章，整个过程让我感觉，这个方向是值得继续尝试的。

现在这个博客想写什么

整理完结构之后，后面写作的方向也比之前清楚了很多。

目前我给自己定下来的分类主要是：

  • C++
  • Linux
  • Agent
  • 音视频
  • 思考

它们基本对应着我现在的技术栈和兴趣重心。

一方面，我会继续写技术学习过程里的东西，比如 C++、Linux、音视频、Agent 相关内容。另一方面，我也希望保留一块“思考”的空间，用来放一些没那么像教程、但依然值得记录下来的判断、复盘和阶段性想法。

我不想把这里做成一个只有“知识点总结”的站，也不想让它变成纯流水账。更理想的状态，是技术和思考都能慢慢沉淀下来。

重新开始，比一直拖着更重要

很多时候一个博客放久了，最容易出现的状态就是：

  • 想改，但一直没动
  • 想写，但总觉得还没整理好
  • 想做成自己的样子，但每次只改一点又停下

最后它就一直停在一个“半成品”状态里。

所以这次我反而觉得，最重要的不是一开始就做到多完美，而是先把它重新启动起来。

主题可以慢慢调，分类和标签可以继续打磨，文章风格也会在后面慢慢稳定。但只要第一篇重新发出来，后面的事情就会顺很多。

从这个角度说，这篇文章本身也是一个标记：

博客重新开始了。

而且这次不是随便试试，而是希望它能真正成为一个长期积累、持续生长的地方。

写在最后

这篇文章没有讲具体技术细节，也不是一篇完整教程。

它更像一个起点记录，用来说明一件事：过去那个比较松散的博客，已经开始重新被整理起来了；而在这个过程中，我也第一次比较完整地把 OpenClaw 接进了自己的博客工作流。

后面大概还会继续写很多更具体的东西，比如：

  • C++ 和内存管理
  • Linux 学习与系统理解
  • Agent 相关的实践与思考
  • 音视频方向的一些笔记
  • 以及一些更个人化的阶段性记录

如果一件事情值得花时间去做，那它大概也值得被认真写下来。
